# 数据集分析文档

## 1. 数据集概述

本项目包含三个主要数据集，用于构建金融投资领域的知识图谱：

1. **公司数据集 (company_data.csv)** - 包含约5千条公司记录
2. **投资事件数据集 (investment_events.csv)** - 包含约2千条投资事件记录
3. **投资结构数据集 (investment_structure.csv)** - 包含约1千条投资机构记录

## 2. 数据集详细分析

### 2.1 公司数据集 (company_data.csv)

**记录数量：** 约5千条

**字段结构：**
- 公司名称
- 英文名称
- 公司简介
- 行业分类
- 成立时间
- 注册资本
- 地区
- 公司类型

**数据特点：**
- 包含公司基本信息和描述性内容
- 部分公司有英文对应名称
- 行业分类多样，涵盖多个领域
- 地理分布广泛

### 2.2 投资事件数据集 (investment_events.csv)

**记录数量：** 约2千条

**字段结构：**
- 事件资讯
- 投资方
- 融资方
- 融资时间
- 轮次
- 金额

**数据特点：**
- 投资方字段可能包含多个投资机构，以空格分隔
- 融资轮次多样化：种子轮、天使轮、Pre-A轮、A轮、A+轮、B轮、C轮、D轮、战略融资、并购、新三板等
- 投资金额格式多样，包含"未披露"、"数百万人民币"、"过千万人民币"等描述性表达
- 时间跨度从2011年至2019年
- 部分记录投资方为"未披露机构"

### 2.3 投资结构数据集 (investment_structure.csv)

**记录数量：** 约1千条

**字段结构：**
- 机构名称
- 介绍
- 行业
- 规模
- 轮次

**数据特点：**
- 包含投资机构的详细信息
- 行业字段描述机构投资的行业分布，如"企业服务36家 文娱内容游戏9家 金融9家"等
- 规模字段部分有具体金额，如"5000万人民币"、"30亿人民币"，部分为空
- 轮次字段描述机构投资的轮次分布，如"A轮56家、B轮47家、C轮36家"等
- 部分记录介绍字段为"暂无信息"

## 3. 数据质量分析

### 3.1 完整性
- 公司数据集：大部分字段完整，但部分公司缺少英文名称或详细描述
- 投资事件数据集：投资金额字段有较多"未披露"记录
- 投资结构数据集：规模字段有较多空值，介绍字段部分缺失

### 3.2 一致性
- 公司名称在不同数据集中可能存在不一致（如全称与简称）
- 投资机构名称可能存在多种表达方式
- 行业分类标准不统一

### 3.3 准确性
- 投资金额表达方式多样，需要标准化处理
- 时间格式相对统一，但需要验证有效性
- 部分投资方为"未披露机构"，需要特殊处理

## 4. 实体与关系抽取要点

### 4.1 核心实体类型
1. **公司实体**：从公司数据集和投资事件数据集中抽取
2. **投资机构实体**：从投资事件和投资结构数据集中抽取
3. **投资事件实体**：从投资事件数据集中抽取

### 4.2 核心关系类型
1. **投资关系**：(投资机构) -[投资]-> (公司)
2. **投资事件关系**：(投资事件) -[涉及投资方]-> (投资机构)
3. **投资事件关系**：(投资事件) -[涉及融资方]-> (公司)
4. **行业分类关系**：(公司) -[属于行业]-> (行业)

### 4.3 实体属性
1. **公司属性**：名称、英文名称、简介、行业、成立时间、注册资本、地区、类型
2. **投资机构属性**：名称、介绍、投资行业偏好、投资规模、投资轮次偏好
3. **投资事件属性**：事件描述、投资时间、投资轮次、投资金额

## 5. LLM抽取重点

基于数据集分析，LLM应重点从以下文本中抽取信息：

1. **事件资讯字段**：抽取投资关系、投资事件详情
2. **公司简介字段**：抽取公司业务描述、行业特征
3. **投资机构介绍字段**：抽取投资偏好、投资策略

避免对结构化数据（如金额、时间、轮次等）进行不必要的LLM处理，这些应通过规则直接解析。

## 6. 数据预处理建议

1. **名称标准化**：建立公司和投资机构的别名映射表
2. **金额标准化**：将各类金额表达转换为统一格式
3. **时间规范化**：统一时间格式，处理异常值
4. **行业分类统一**：建立行业分类标准，映射各类行业表达

## 7. 批量处理策略

基于数据集特点，建议以下批量处理策略：

1. **按事件类型分组**：将相似的投资事件合并处理
2. **按文本长度分组**：控制每批处理的token总数
3. **按投资机构分组**：将同一投资机构的多个事件合并处理
4. **按行业分组**：将同行业的公司描述合并处理

以上分析将作为LLM增强标注模块设计的基础依据。
